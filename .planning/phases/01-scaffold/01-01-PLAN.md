**Step 6 — Write `src/stores/ui.ts`** (dark mode store — needed by App.vue)

```typescript
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useUIStore = defineStore('ui', () => {
  const darkMode = ref(false)

  function initDarkMode() {
    darkMode.value = !!localStorage.getItem('travelapp_dark')
    document.documentElement.classList.toggle('dark', darkMode.value)
  }

  function toggleDark() {
    darkMode.value = !darkMode.value
    document.documentElement.classList.toggle('dark', darkMode.value)
    localStorage.setItem('travelapp_dark', darkMode.value ? '1' : '')
  }

  return { darkMode, initDarkMode, toggleDark }
})
```

**Step 1 — Write `tailwind.config.js`**

```javascript
/** @type {import('tailwindcss').Config} */
export default {
```

**Step 4 — Update `vercel.json`** (merge new SPA rewrite with existing photo proxy rule)

The existing `vercel.json` has a storage proxy for `/photos/:path*`. The SPA rewrite must be added but must NOT intercept the photos proxy. Order matters in Vercel rewrites — more specific rules first, catch-all last:

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "rewrites": [
    {
      "source": "/photos/:path*",
      "destination": "https://xedunxhvrhkyphcnllxa.supabase.co/storage/v1/object/public/trip-photos/:path*"
    },
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

CRITICAL: The photos proxy rule MUST come before the SPA catch-all. If the SPA rewrite is first, all `/photos/...` requests get routed to `index.html` instead of Supabase Storage.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Supabase | All DB/storage calls go over HTTPS from the client; no server-side backend |
| localStorage → App | Dark mode preference and trip ID read back from localStorage; no auth data stored here |
| Public source → Credentials | The existing CDN index.html contains the anon key in plaintext source history |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Supabase anon key in `index-cdn-backup.html` | mitigate | Rotate the Supabase anon key after Phase 1 completes (key is in git history from CDN era); new key goes only in `.env.local` (gitignored). Document this in the plan summary. |
| T-01-02 | Information Disclosure | `.env.local` accidentally committed | mitigate | `.gitignore` must include `.env.local` and `.env.*.local`; verify `git status` shows `.env.local` as untracked before first commit |
| T-01-03 | Tampering | `vercel.json` SPA rewrite order | mitigate | Photos proxy rule listed first so storage URLs are not intercepted by the SPA catch-all; ordering verified in acceptance criteria |
| T-01-04 | Elevation of Privilege | Supabase anon key scoped to `anon` role | accept | Anon key is intentionally public-facing; Supabase RLS (not client-side) enforces data access; acceptable risk for Phase 1 scaffold |
| T-01-05 | Spoofing | localStorage trip ID can be overwritten by any script | accept | No PII in trip ID; trip data access controlled by Supabase RLS in later phases; acceptable for Phase 1 |
